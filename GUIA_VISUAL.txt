┃ 🔄 CICLO DE VIDA DO AGENDAMENTO                                          ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

   1. CRIAR AGENDAMENTO
      ├─ Usuário preenche: Cliente, Profissional, Serviço, Data/Hora
      ├─ Sistema verifica conflito de horário
      ├─ Sistema obtém preço do serviço
      └─ Agendamento é criado com status "agendado"
   
   2. CONFIRMAR AGENDAMENTO
      ├─ Usuário clica [✓ Confirmar] na agenda
      ├─ Status muda para "confirmado"
      └─ Agendamento aparece no dashboard
   
   3. CANCELAR AGENDAMENTO
      ├─ Usuário clica [✕ Cancelar] na agenda
      ├─ Status muda para "cancelado"
      └─ Não conta mais para relatórios
   
   4. VISUALIZAR LUCROS
      ├─ Dashboard filtra agendamentos confirmados
      ├─ Agrupa por serviço (quanto cada um faturou)
      ├─ Agrupa por profissional (quanto cada um ganhou)
      └─ Mostra ticket médio
